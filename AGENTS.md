# Flask Web アプリ開発ルール

## 変更してはいけないファイル

- `.htaccess` と `index.cgi` は Web サーバ上で Flask アプリを動作させるための設定ファイルである。
- これらのファイルは**一切変更しない**。上書き、削除、整形、パーミッション変更も行わない。

## アプリケーションとデータベース

- `app.py` は作成する Web アプリに合わせて書き換えまたは置き換えてよい。
- サンプルのデータベース接続・切断などの補助関数は、必要に応じて流用してよい。
- 既存の `database.db` は削除し、アプリ用に作成した SQLite データベースファイルを配置する。
- データベース名を変更する場合は、`app.py` 内の接続先も必ず一致させる。

## テンプレートと静的ファイル

- `templates/` 配下の HTML は、作成する Web アプリ用のテンプレートへ置き換える。
- 静的ファイルは `static/` 配下に格納する。スタイルシートは `static/css/style.css`、JavaScript は `static/js/app.js` を使用する。

## Web アプリ要件：AIモデル・リーダーボード検索

### 目的

SQLite に保存した AI モデルの公開リーダーボードを、Web ブラウザで検索・比較・集計できるようにする。既存の正規化済みテーブルと `leaderboard_result_view` を活用し、レポートで示した検索要件を画面上で満たす。

### 使用データ

- データベースファイルは `leaderboard.db` を使用する。
- 主なテーブルは `organizations`、`licenses`、`models`、`arenas`、`leaderboard_results` である。
- 一覧表示では、これらを結合したビュー `leaderboard_result_view` を優先して使用する。

### 機能要件

- 評価対象（arena）を指定し、その評価対象における AI モデルの順位一覧を表示できる。
- 一覧は少なくとも順位、モデル名、開発組織、ライセンス、カテゴリ、公開日、レート、レート下限・上限、投票数を表示する。
- 開発組織を指定し、その組織が公開したモデルの評価結果一覧を表示できる。
- ライセンスを指定し、そのライセンスに対応するモデルの一覧を表示できる。
- AI モデルを指定し、そのモデルが掲載されている評価対象と評価結果の一覧を表示できる。
- 開発組織ごとに、掲載モデル数、平均レート、最高レートを集計して表示できる。

### 推奨画面・URL構成

- `/`：評価対象・組織・ライセンスなどで絞り込めるランキング一覧画面。順位順で表示する。
- `/models/<model_id>`：モデルの基本情報と、評価対象ごとの結果を表示する詳細画面。
- `/organizations/<organization_id>`：組織のモデル一覧、評価結果、モデル数・平均レート・最高レートを表示する詳細画面。
- `/licenses/<license_id>`：ライセンスに対応するモデル一覧を表示する画面。

### 実装上の要件

- 検索・絞り込みには GET パラメータを使用し、URL を共有・再読み込みしても同じ結果を表示できるようにする。
- ランキング一覧は1ページ50件まで表示し、ページ番号と前後移動を用いて続きの結果を参照できるようにする。ページ番号は先頭・末尾・現在ページ付近を表示し、それ以外を省略記号で表す。
- SQL は必ずプレースホルダを使って値を渡し、文字列連結で検索条件を組み立てない。
- データ参照が主目的のアプリとし、既存データを更新・削除する機能は実装しない。
- データがない検索条件でも、サーバーエラーではなく「該当するデータはありません」と表示する。

## パーミッション

- ディレクトリ、`index.cgi`、`app.py` は `755` (`rwxr-xr-x`) にする。
- それ以外の通常ファイルは `644` (`rw-r--r--`) にする。
- 本番の Web サーバ／Flask は別アカウントで実行されるため、所有者だけが読み取り・実行できる設定にしない。
